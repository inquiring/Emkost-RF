# Project Context

Проект: одностраничный лендинг `Emkost-RF` на чистом `HTML + CSS + JS` с локальным `Node.js + Express` API в [server.js](/Users/alexander/Documents/Emkost-RF/server.js).

## Важный контекст

- Сайт продает только `б/у` ёмкости, резервуары, цистерны и бочки.
- В публичных текстах использовать написание `ёмкость` / `ёмкости` через `ё`.
- Базовый диапазон объемов на сайте: `от 5 до 160 м³`.
- По странице нужно явно поддерживать сценарии: `вода`, `ГСМ`, `химия`, `септик`, `газ / задачи под давлением`.
- Без отдельного запроса не менять визуальную структуру лендинга, сетку секций и логику форм.

## Формы и заявки

- Основной endpoint формы: `POST /api/create-order`.
- Обязательные поля формы:
  - `name`
  - `phone`
  - `volume`
  - `purpose`
  - `delivery`
  - `consent`
- Поле `task` необязательное, но должно сохраняться и передаваться в уведомления.
- Телефон валидируется в формате `+7 (999) 123-45-67`.
- Формы на фронтенде отправляются через `fetch` из [assets/js/main.js](/Users/alexander/Documents/Emkost-RF/assets/js/main.js).

## Хранение лидов

- Сервер сначала сохраняет заявку в локальную `SQLite` базу, затем отправляет уведомление в Telegram.
- Путь к БД задается через `DATABASE_PATH`.
- Файл БД не должен попадать в git.
- Telegram не является единственным местом хранения заявки.

## Доступ к лидам

- `GET /api/leads` — защищенный JSON-список заявок.
- `GET /api/leads.csv` — защищенная CSV-выгрузка заявок.
- Доступ к этим endpoint должен быть только через `ADMIN_TOKEN`.

## Переменные окружения

- `TELEGRAM_BOT_TOKEN`
- `TELEGRAM_TARGET_CHAT`
- `ADMIN_TOKEN`
- `DATABASE_PATH`
- `PORT`

## SEO и публичные ссылки

- В `head` уже настроены базовые `SEO`, `Open Graph` и `Twitter Card` метатеги.
- Сейчас `canonical` и `og:url` смотрят на Render preview URL.
- При переходе на боевой домен нужно обновить:
  - `canonical`
  - `og:url`
  - при необходимости `og:image`

## Git hygiene

- Не коммитить `.env.local`, `node_modules`, файлы SQLite и временные выгрузки.
- Перед пушем проверять `git status` и `git diff --cached`.

## Юридический и инфраструктурный контекст

- Для боевого запуска с заявками граждан РФ предполагается размещение сервера и хранения данных на территории РФ.
- Render допустим как preview, но не считается финальной целевой инфраструктурой под это требование.
